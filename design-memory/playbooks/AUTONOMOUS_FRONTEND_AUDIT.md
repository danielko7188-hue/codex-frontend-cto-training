# Autonomous Frontend Audit

This is the operating promise: the founder should not need to supply screenshots for routine UX/UI work.

## Default behavior

When asked to improve a frontend, Codex should:

1. inspect the repo
2. run the app
3. find the important screens
4. capture desktop and mobile screenshots
5. critique the current UI
6. redesign and verify

## When to ask the founder for input

Ask only when:

- the product direction itself is unclear
- two design directions have materially different business implications
- the founder needs to make a brand-level choice

Do not ask just because the current page has not been screenshotted for you.

## Minimum audit questions

- What is the page trying to get the user to do?
- Does the current hierarchy support that goal?
- Does it feel premium or generic?
- Does it create trust?
- Does it survive on mobile?

## Definition of a useful audit

A useful audit names:

- what is weak
- why it is weak
- what better direction to take
- what changed after implementation
