# Reference Bank

This bank is for structured references, not inspiration hoarding.

Use it to store examples in three buckets:

- [love](./love)
- [hate](./hate)
- [neutral](./neutral)

## How to add an entry

1. Copy [reference-entry-template.md](../templates/reference-entry-template.md).
2. Save it in the right bucket.
3. Name it after the product or reference.
4. Fill in the reusable lesson.

## Standard

A good entry explains:

- what the reference is
- what feels strong or weak
- what to borrow
- what not to borrow
- when the pattern is appropriate

If an entry does not create a reusable design rule, it is not done.
