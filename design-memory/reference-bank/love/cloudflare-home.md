# Reference Entry

- Name: Cloudflare
- URL or source: https://www.cloudflare.com/
- Product type: enterprise infrastructure / security / developer platform
- Bucket: love
- Date reviewed: 2026-04-07

## First impression

- Strong: authoritative, technical, high-trust, proof-heavy without feeling stale
- Weak: can feel too broad and corporate if copied onto a smaller product

## What to borrow

- messaging: simple three-part framing like connect, protect, build
- proof: large network and threat numbers create credibility fast
- CTA structure: free start plus guided demo supports both self-serve and sales-led motions
- layout: broad platform story still stays organized by outcome, not feature soup
- tone: technical confidence without sounding robotic

## What not to borrow

- huge platform breadth when the product only solves one narrow problem
- enterprise jargon without matching operational proof

## Why this matters

- It is a strong example of enterprise trust that still feels modern and product-led.

## Reusable lesson

- When the product sells reliability, numbers and structure should carry authority before visual flair does.
