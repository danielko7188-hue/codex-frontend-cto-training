# Pattern Reference Pack 2026-04-07

This file maps common UI patterns to stronger real-world references.

Its purpose is execution quality.

Style lanes answer "how should this feel?"

Business-type packs answer "what kind of company is this?"

Screen-type packs answer "what is this screen trying to do?"

Pattern packs answer "what should this specific section or component look like and how should it behave?"

Use this pack after choosing the business type, style lane, and screen type.

## Hero Section

- Primary anchors: [Vercel](../reference-bank/love/vercel.md), [Linear](../reference-bank/love/linear.md), [ServiceTitan](../reference-bank/love/servicetitan-home.md)
- Best for: homepages, launches, product intros, service entry pages
- Borrow: one strong promise, one strong proof move, one obvious action
- Watch out for: oversized copy with no grounding, fake premium emptiness

## Proof Band / Trust Strip

- Primary anchors: [Cloudflare](../reference-bank/love/cloudflare-home.md), [Ramp](../reference-bank/love/ramp-home.md), [ServiceTitan](../reference-bank/love/servicetitan-home.md)
- Best for: social proof, logos, stats, guarantees, platform reach
- Borrow: fast credibility, numbers with meaning, tight spacing, low-friction scanning
- Watch out for: meaningless vanity stats or logo walls with no story

## Process / How It Works Section

- Primary anchors: [Notion Sites](../reference-bank/love/notion-sites.md), [Airbnb Host Homes](../reference-bank/love/airbnb-host-homes.md), [One Medical](../reference-bank/love/one-medical-home.md)
- Best for: onboarding pages, service pages, setup flows, guided products
- Borrow: short sequence, low-anxiety steps, realistic expectations
- Watch out for: too many steps, clever labels, or hidden friction

## Case Study / Testimonial Block

- Primary anchors: [ServiceTitan](../reference-bank/love/servicetitan-home.md), [Mercury](../reference-bank/love/mercury.md), [Ramp](../reference-bank/love/ramp-home.md)
- Best for: services, B2B sales, trust-first products, lead-gen pages
- Borrow: specific outcomes, named customers, concise storytelling, visible before/after impact
- Watch out for: generic praise that says nothing concrete

## Comparison Table / Plan Matrix

- Primary anchors: [Stripe](../reference-bank/love/stripe-home.md), [Notion](../reference-bank/love/notion-home.md), [Shopify Polaris](../reference-bank/love/shopify-polaris.md)
- Best for: pricing, product comparisons, plan choice, capability breakdowns
- Borrow: clear row labels, visible plan differences, default recommendation, readable density
- Watch out for: hiding the real difference between plans

## Dashboard Shell

- Primary anchors: [Retool](../reference-bank/love/retool-home.md), [Cal Teams](../reference-bank/love/cal-teams.md), [Intercom Inbox](../reference-bank/love/intercom-inbox.md)
- Best for: SaaS apps, internal tools, operator consoles, AI workspaces
- Borrow: clear nav hierarchy, useful default canvas, visible task areas, calm density
- Watch out for: big empty cards and wasted horizontal space

## Sidebar / Command Navigation

- Primary anchors: [Claude Code](../reference-bank/love/claude-code.md), [Shopify Polaris](../reference-bank/love/shopify-polaris.md), [Cloudflare](../reference-bank/love/cloudflare-home.md)
- Best for: multi-surface apps, settings-heavy products, AI tools, admin consoles
- Borrow: grouping by job, obvious current state, quiet secondary items, safe destructive placement
- Watch out for: flat nav lists with no hierarchy

## Data Table / Results List

- Primary anchors: [Cloudflare](../reference-bank/love/cloudflare-home.md), [Retool](../reference-bank/love/retool-home.md), [Shopify Polaris](../reference-bank/love/shopify-polaris.md)
- Best for: records, queue views, reporting, audits, approvals, operations
- Borrow: strong row hierarchy, useful status chips, sortable columns, calm density
- Watch out for: styling tables like marketing cards

## Empty State

- Primary anchors: [Notion](../reference-bank/love/notion-home.md), [Cal Teams](../reference-bank/love/cal-teams.md), [Intercom Inbox](../reference-bank/love/intercom-inbox.md)
- Best for: first-run surfaces, no-data states, newly created accounts, zero-results pages
- Borrow: one next step, short explanation, visible value of filling the state
- Watch out for: overly cute empties that do not help the user move

## Activity Feed / Timeline

- Primary anchors: [Intercom Inbox](../reference-bank/love/intercom-inbox.md), [Cloudflare](../reference-bank/love/cloudflare-home.md), [Cal Teams](../reference-bank/love/cal-teams.md)
- Best for: support tools, approval histories, operational workflows, audit logs
- Borrow: compact chronology, strong timestamps, obvious actor/action/result
- Watch out for: walls of text with no scannable structure

## Chat Composer

- Primary anchors: [ChatGPT Overview](../reference-bank/love/chatgpt-overview.md), [Claude Code](../reference-bank/love/claude-code.md), [OpenAI Codex](../reference-bank/love/openai-codex.md)
- Best for: AI chat, copilots, command surfaces, prompt-based tools
- Borrow: clear prompt affordance, obvious submit state, room for attachments or tools, visible mode/context
- Watch out for: input bars that hide power but still confuse beginners

## Tool Result Card / Artifact View

- Primary anchors: [Claude Code](../reference-bank/love/claude-code.md), [Retool](../reference-bank/love/retool-home.md), [OpenAI Codex](../reference-bank/love/openai-codex.md)
- Best for: AI tool calls, generated outputs, workflow artifacts, structured results
- Borrow: title, source, status, next action, inspectable details
- Watch out for: pretty blobs with no actionable follow-up

## Approval Panel

- Primary anchors: [Claude Code](../reference-bank/love/claude-code.md), [Cloudflare](../reference-bank/love/cloudflare-home.md), [Retool](../reference-bank/love/retool-home.md)
- Best for: AI review steps, admin approvals, risky actions, compliance checkpoints
- Borrow: obvious choices, consequence language, diff or summary visibility, auditability
- Watch out for: “approve” buttons without enough context

## Settings Group

- Primary anchors: [Shopify Polaris](../reference-bank/love/shopify-polaris.md), [Cloudflare](../reference-bank/love/cloudflare-home.md), [Mercury](../reference-bank/love/mercury.md)
- Best for: preferences, team settings, permissions, billing, security
- Borrow: strong grouping, explanatory helper text, safe destructive boundaries, predictable controls
- Watch out for: over-designed settings that make basic tasks slower

## Lead Form

- Primary anchors: [ServiceTitan](../reference-bank/love/servicetitan-home.md), [Airbnb Host Homes](../reference-bank/love/airbnb-host-homes.md), [One Medical](../reference-bank/love/one-medical-home.md)
- Best for: quotes, demos, consults, memberships, inbound capture
- Borrow: short fields, trust nearby, visible outcome, easy mobile completion
- Watch out for: too many required inputs before trust is earned

## How To Use This Pack

1. Choose the business type with [BUSINESS_TYPE_REFERENCE_PACK_2026-04-07.md](./BUSINESS_TYPE_REFERENCE_PACK_2026-04-07.md).
2. Choose the style lane with [STYLE_SPECTRUM_2026-04-07.md](./STYLE_SPECTRUM_2026-04-07.md).
3. Choose the screen type with [SCREEN_TYPE_REFERENCE_PACK_2026-04-07.md](./SCREEN_TYPE_REFERENCE_PACK_2026-04-07.md).
4. Choose the 2-4 patterns that matter most on that screen.
5. Write down what each pattern must accomplish before coding it.

## Core Rule

Generic UI often comes from using the wrong pattern, not just the wrong colors.
