## How to use this pack

- Use the visual [reference-bank](../reference-bank/README.md) when the question is "what should this feel like?"
- Use this code set when the question is "how should we structure or implement this well?"
- Use both when the goal is a premium UI that also behaves like a serious product.

## Recommended reading order

1. `shadcn/ui` for composition and product-speed pragmatism
2. `Radix Primitives` for accessibility and interaction foundations
3. `Primer React` for operational seriousness and dense app surfaces
4. `Vercel AI Chatbot` and `assistant-ui` for AI workspace patterns

## What this pack teaches especially well

- how to compose reliable primitives into product UI
- how to keep tokens and themes disciplined
- how to structure dense apps without collapsing into clutter
- how to build AI chat surfaces that feel like tools, not toys
- how to start new products from stronger official templates instead of weak blank-page defaults
- how to translate the feel of a world-class product into reusable DESIGN.md rules for future builds

## Important limit

These codebases can teach implementation quality and system thinking.

They do not automatically teach brand taste, section pacing, trust signals, or premium visual restraint on their own.
