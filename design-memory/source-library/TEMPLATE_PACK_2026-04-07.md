# Template Pack: 2026-04-07

This is the Phase 2 training pack for official starter templates and example apps.

Use this when the problem is not just "how should this component work?" but also "what is a strong starting skeleton for this kind of product?"

## Best for SaaS and dashboard starters

## Best for elegant React and Next.js starter structure

- [Mantine Next App Template](./templates/mantine-next-app-template.md)

## How to use this pack

- Use these starters to study app shell decisions, route structure, auth and dashboard scaffolding, and content sequencing.
- Borrow structure, not surface-level sameness.
- If the starter feels generic, reshape it to match the founder taste profile before shipping.

## Recommended order

1. `Next.js SaaS Starter` for modern SaaS information architecture
2. `Vercel Admin Dashboard Template` for authenticated dashboard structure
3. `Mantine Next App Template` for clean setup and maintainable starter ergonomics
4. `Vercel Templates Hub` and `Next.js Official Examples` when you need breadth

## Important limit

A template is a starting point, not proof of strong taste.

Templates are most useful when they save structure work while the final hierarchy, copy, proof, spacing, and visual restraint still come from the founder taste system and visual reference bank.
